# Screenshot 1

## Dashboard iGracias

### Design Pattern yang Digunakan

* Navigation Bar (menu di sebelah kiri)
* Site Branding (logo iGracias dan Telkom University)
* Dashboard Layout
* Search Menu

### Hal yang Kurang Baik

1. Terlalu banyak menu pada sidebar kiri

   * Pengguna baru bisa kebingungan.
   * Perlu scrolling panjang untuk menemukan fitur tertentu.

2. Banner terlalu mendominasi layar

   * Informasi akademik penting tertutup oleh banner promosi event.

3. Hierarki informasi kurang jelas

   * Informasi yang penting (misalnya tagihan, nilai, jadwal) tidak langsung terlihat.
   * Yang pertama dilihat justru iklan acara.

### Think Aloud

> Ketika membuka iGracias, saya ingin melihat informasi akademik saya. Namun perhatian saya justru tertuju pada banner Market Day yang sangat besar. Saya harus mencari menu yang saya butuhkan di sidebar yang cukup panjang.

### Rekomendasi

* Kurangi ukuran banner.
* Tambahkan Quick Access:

  * Jadwal
  * Nilai
  * Tagihan
  * Presensi

# Screenshot 2

## Digital KTM

### Design Pattern yang Digunakan

* Card Layout
* Process Funnel sederhana
* Site Branding

### Hal yang Kurang Baik

1. Banyak ruang kosong (white space)

   Hampir 70% halaman kosong.

2. Tidak ada tombol aksi utama

   Pengguna mungkin bertanya:

   > "Lalu saya harus ngapain?"

3. Informasi kurang lengkap

   Tidak ada tombol:

   * Download KTM
   * Simpan QR
   * Cetak KTM

### Think Aloud

> Saya berhasil membuka KTM digital, tetapi saya tidak melihat opsi untuk mengunduh atau menyimpan KTM tersebut. Saya tidak yakin bagaimana cara menggunakannya selain melakukan screenshot.

### Rekomendasi

Tambahkan tombol:

* Download PDF
* Download QR Code
* Print KTM

# Screenshot 3

## Upload Competition For Student

Ini menurutku screenshot terbaik untuk tugas karena banyak masalah UX.

### Design Pattern yang Digunakan

* Form Upload
* Navigation Sidebar
* Data Table

### Hal yang Kurang Baik

1. Form terlalu polos

   Tidak ada petunjuk langkah-langkah pengisian.

2. Tombol Upload Data tidak menonjol

   Sulit dibedakan dari elemen lain.

3. Validasi kurang jelas

   Hanya tertulis:

   > File harus berformat pdf maks 3MB

   tetapi tidak ada indikator apakah file yang dipilih sudah benar.

4. Tabel kosong tanpa penjelasan

   Muncul:

   > No data available in table

   tetapi tidak dijelaskan apa yang harus dilakukan.

### Think Aloud

> Saya ingin mengunggah sertifikat kompetisi. Saya melihat beberapa kolom yang harus diisi, tetapi saya tidak langsung memahami alur prosesnya. Tombol upload juga kurang menonjol sehingga saya harus memperhatikannya lebih lama.

### Rekomendasi

* Tambahkan progress indicator.
* Buat tombol upload lebih besar.
* Tambahkan preview file.
* Tambahkan status:

  * Draft
  * Submitted
  * Approved
  * Rejected

# Dark Pattern

Menurutku dari ketiga screenshot ini:

**Tidak ditemukan dark pattern yang disengaja.**

Yang ditemukan adalah:

* Poor Visual Hierarchy
* Information Overload
* Inefficient Navigation
* Poor Visibility of System Status

Jadi di laporan bisa tulis:

> Berdasarkan hasil observasi, tidak ditemukan dark pattern seperti hidden cost, confirm shaming, maupun bait and switch. Namun ditemukan beberapa masalah usability yang dapat mengurangi kenyamanan pengguna dalam menggunakan sistem.
